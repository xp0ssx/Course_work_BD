# Сервис обзоров игр

Это мой учебный проект — простой сервис для публикации обзоров и комментариев к играм. Интерфейс сделан на Streamlit, данные хранятся в PostgreSQL. Для ускорения и уведомлений в проекте использовался Redis (опционально).

## Что в репозитории

- `app.py` — Streamlit-приложение.
- `db/` — операции работы с базой: подключение и CRUD для пользователей, игр, обзоров и комментариев.
- `table/schema.sql` — SQL-дамп со схемой базы (таблицы, последовательности, триггер проверки контента).
- `redis_implementation_report.md` — описание, как я использовал Redis (кеширование и pub/sub).

## Функционал

- Регистрация и вход пользователей (bcrypt для хеширования паролей).
- Добавление и поиск игр.
- Добавление обзоров и комментариев.
- В БД есть триггер, который блокирует отзывы с запрещёнными словами.

## Быстрая установка

1. Создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install streamlit psycopg2-binary bcrypt redis
```

Если на Linux возникают ошибки при установке `psycopg2-binary`, установите системные зависимости:

```bash
sudo apt update && sudo apt install -y build-essential libpq-dev
```

## Создание базы данных

Пример команд для создания базы и применения схемы:

```bash
createdb dbcw
psql -d dbcw -f table/schema.sql
```

В `db/db_connection.py` сейчас прописаны примерные значения подключения. Лучше использовать переменные окружения. Например:

```bash
export DB_HOST=localhost
export DB_NAME=dbcw
export DB_USER=myuser
export DB_PASSWORD='secret'
```

## Запуск

```bash
streamlit run app.py
```

Откройте адрес, который выдаст Streamlit (обычно http://localhost:8501).

## Redis (опционально)

Я использовал Redis для кеширования популярных игр, хранения сессий и pub/sub-уведомлений. Если хотите воспользоваться Redis — установите и запустите его и добавьте параметры подключения в проект.

## Замечания по безопасности

- Не храните реальные пароли и секреты в коде — используйте переменные окружения.
- Пароли пользователей хранятся в виде bcrypt-хэша.

## Что можно улучшить

- Вынести конфигурацию в переменные окружения или файл `.env`.
- Добавить `requirements.txt` и тесты (pytest).
